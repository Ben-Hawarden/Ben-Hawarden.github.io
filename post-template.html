Overview

Headless is a medium-difficulty Linux box on Hack The Box. It runs a web application vulnerable to blind XSS, which leads to cookie theft and ultimately remote code execution. Privilege escalation involves exploiting a misconfigured syscheck script.

Difficulty: Medium | OS: Linux | Release: 2026-03-15

Reconnaissance

Starting with an nmap scan to identify open ports and services.

nmap -sC -sV -oN scan.txt 10.10.11.XX

PORT     STATE SERVICE VERSION
22/tcp   open  ssh     OpenSSH 8.9
5000/tcp open  http    Werkzeug/2.2.2

Two ports open: SSH on 22 and a Python Werkzeug web server on 5000.

Enumeration

Navigating to port 5000 reveals a simple web application with a contact form. Running directory enumeration with ffuf to find additional endpoints.

ffuf -u http://10.10.11.XX:5000/FUZZ -w /usr/share/wordlists/dirb/common.txt

/dashboard            [Status: 500]
/support              [Status: 200]

The /dashboard endpoint returns a 500 without authentication. The /support page has the contact form we saw.

Exploitation

The support form is vulnerable to blind XSS. When an admin reviews submitted tickets, our injected script executes in their browser context, leaking their session cookie.

With the admin cookie, we can access /dashboard which has a system command injection vulnerability in the date parameter.

Privilege Escalation

Checking sudo permissions reveals we can run a syscheck script as root.

sudo -l
User dvir may run the following commands:
    (ALL) NOPASSWD: /usr/bin/syscheck

The script calls a relative path to another script, which we can hijack by creating our own version in the current directory.

Flags

User and root flags obtained. Box complete.

Takeaways

  • Blind XSS can be just as dangerous as reflected XSS — always test for it
  • Cookie theft through XSS can bypass authentication entirely
  • Scripts running as root should always use absolute paths
  • Check sudo permissions early in your privilege escalation workflow
